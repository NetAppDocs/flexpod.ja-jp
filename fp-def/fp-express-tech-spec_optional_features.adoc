このセクションでは、 FlexPod Express のオプション機能について説明します。


NOTE: SAN ブートはオプションですが、ハードウェアのアップグレードや交換時にサーバー プロファイルを移動できるため、強く推奨されます。



== iSCSI ブートオプション

iSCSIブートオプションの最小要件は以下のとおりです：

* ネットアップストレージコントローラでアクティブ化された iSCSI ライセンス / 機能
* ネットアップストレージコントローラ HA ペアの各ノードに、 2 ポート 10Gbps イーサネットアダプタを搭載します
* iSCSI ブートに対応した Cisco UCS サーバ内のアダプタ




== Cisco UCS X シリーズ ダイレクトおよびダイレクトアタッチ AFF/ASA/FAS オプション

NetApp ストレージ コントローラは、上流の SAN スイッチを必要とせずに、Cisco UCS ファブリック インターコネクトに直接接続できます。FlexPod Express は、UCS X-Series Direct との FC ブートおよび iSCSI ブートの両方をサポートしています。

ライセンスとハードウェアの要件は次のとおりです。

* ネットアップストレージコントローラにはプロトコルライセンスが必要です。
* サーバには Cisco UCS アダプタ（イニシエータ）が必要です。サポートされている Cisco UCS アダプタの一覧については、ネットアップを参照してください http://mysupport.netapp.com/matrix["IMT"^]。
* ネットアップストレージコントローラにはターゲットアダプタが必要です。


次の図は、IPベースSAN向けのUCS X-Series Directおよび直接接続されたASAトポロジーを示しています。

image:fp-express-tech-spec_image10.png["UCS X-Series ダイレクトおよびダイレクトアタッチ ASA IPベース SAN のトポロジー"]

* 注： *

* Cisco UCS はイーサネットスイッチングモードで設定されます。
* ターゲットからファブリックインターコネクトへの iSCSI ポートは、 iSCSI データ用のイーサネットストレージポートとして構成されます。


次の図は、IPベースおよびFCベースのSAN向けのUCS X-Series DirectおよびASA直接接続トポロジーを示しています。

image:fp-express-tech-spec_image11.png["UCS X-Series ダイレクトおよびダイレクトアタッチ ASA の IP ベースおよび FC ベースの SAN トポロジー"]

* 注： *

* Cisco UCS は FC スイッチングモードで設定されます。
* ターゲットからファブリックインターコネクトへの FC ポートは、 FC ストレージポートとして構成されます。


この構成の詳細については、 https://www.netapp.com/pdf.html?item=/media/135828-flexpod-san-ucs-xdirect-asa-nva-deploy-guide.pdf["Cisco UCS X-Series DirectおよびNetApp ASAを使用したFlexPod SAN解決策"^]を参照してください。


NOTE: 上記の例では NetApp ASA ストレージコントローラーを使用していますが、同じ構成は NetApp AFF および FAS ストレージコントローラーでもサポートされています。
